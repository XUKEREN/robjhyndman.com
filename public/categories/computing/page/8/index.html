Converting eps to pdf

1 October 2009

computing, LaTeX

Simply include the package epstopdf. Then when you use pdflatex, the eps files will be automatically converted to pdf at compile time. (The conversion only happens the first time you process the file, and is skipped if there is already a pdf file with the same name.) For example: \documentclass{article} \usepackage{graphicx,epstopdf} \begin{document} \includegraphics[width=\textwidth]{fig1} \end{document} Then even though the only graphics file available is fig1.eps, this will still be processed ok using pdflatex or pdftexify.

Maintaining local LaTeX files

14 August 2009

computing, LaTeX

If you use LaTeX, then you probably have a bib file — a data base of all the papers and books that you have cited. It is much more efficient to keep one database in one location, than have multiple copies of it floating around your hard drive. (Or even worse, have different bib files created for different papers.) You might also have a few of your own style files, and again it is best to keep these in a central location and not have duplicates all over the place.
